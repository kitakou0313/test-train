# テスト分析と設計

Glenford Myers推奨のテスト設計の流れ

- 仕様が入力条件の組み合わせを含んでいる場合は原因-結果グラフの作成から始める
- どんな場合でも境界値分析を行う
- 入出力の有効無効パーティションを見分け、必要ならテストケースを補足
- エラー推測技法によりテストケースを補足
- 要求されるカバレッジに上の手順に満たない場合、ホワイトボックス技法によりテストを追加する

## テスト技法の分類

### ブラックボックス
- テストアイテムやテスト条件に対する要件が明らかになっている前提
- テスト対象の入出力にのみ注目したテスト技法
    - テストケースの網羅度に注目する

### ホワイトボックス
- テスト対象ソフトウェアの内部構造（コード、ドキュメント）が明らかになっている前提
- テスト対象のソフトウェアの内部構造や処理に注目する技法
    - C1, C0などのカバレッジに注目する

### 経験ベースのテスト技法 
- 開発者、テスト担当者に経験がある前提
- 開発者、テスト担当者の経験に基づいてテストケースを作成する
- 最終確認の役割として一定の効果が見込まれる
    - 探索的テスト


ブラックボックス、ホワイトボックスの中間としてグレーボックステスト技法もある コードの詳細よりも荒い粒度でテストする


## ブラックボックステスト技法の詳細
- 同値分割法
    - 同値パーティションごとに代表的な値を一つ選びテストする
        - 同値パーティション -> ソフトウェアへの入力, 出力データのうち同じ処理が行われると想定した集合
        - 例
            - 内部変数
            - 時間関連の値
            - インターフェースのパラメーター
            - 構成アイテム
            - 順序関係の有無、離散 or 連続
        - パーティションはMECEになるように定義する
            - 重複あり -> どのパーティションに対するテストをしたのか不明
            - 漏れあり -> テストケースのもれ
        - 有効パーティション、無効パーティションについても考慮してテストする
            - 無効パーティション -> テスト対象が拒否すべきもの、仕様で定義されていないもの
            - ソフトウェアの内部構造を意識せず、入力されるデータの集合にのみ注目すること
    - カバレッジの算出
        - テストしたパーティション数 / 全パーティション数
- 境界値分析（Boundary Value Analysis）
    - 順序関係があるデータを扱うソフトウェアのテストに用いられる手法
    - 境界値およびその隣接値をテストする手法
        - 境界値 -> 各パーティションに含まれる値のうち、下限又は上限
            - 
        - 条件の境界の値に仕様の考慮もれが集中するため、集中的にテストする手法
    - 種類
        - 前提
            - 「20歳以上, 34歳以下」の年齢のユーザーに対するテストを考える
                - このパーティションF1に属する境界値は「20」と「34」
            - 「35歳以上」を含むパーティションF2が存在している
            - 入力の全集合は20 <= x <= 100の整数とする
        - 2値VBA
            - 境界値にくわえ、各パーティションごと隣接するパーティション内で各境界値に隣接する値をテストする
            - 19, 20, 34, 35をテストする
        - 3値VBA
            - 境界値にくわえ、各パーティションごと隣接するパーティション内で各境界値に隣接する値とパーティション内の値のうち各境界値に隣接する値をテストする
            - 19, 20, 21, 33, 34, 35をテストする
                - F2の境界値を35とした場合、34, 35, 36をテストすることになり、3値VBAとしては不適切
                    - メモ:不適切だと何が良くない？
                - そのため、あくまでも境界値は34のみであり34 < x < 35に関しては仕様を整備した方が良い
    - カバレッジの算出
        - テストした境界値とその隣接値の数 / テストすべき境界値とその隣接値の個数
- デシジョンテーブルテスト
    - カバレッジの算出
- 状態遷移テスト
    - カバレッジの算出
